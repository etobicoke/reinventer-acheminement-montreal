⚜ Live demo · the deterministic resolver, on real requests

Resolve the asset. Route with a reason. Redirect when it isn't ours.

Every location below is a real request from the City's 311 open data, and the routing shown is the resolver's — which matches the City's own decision. The photo/voice step is the input; this page demonstrates the part that makes routing trustworthy: a deterministic, citable resolver that never guesses a department.

A citizen sends a photo and a few words. Perception turns that into an asset + defect. Then a deterministic resolver decides whose problem it is — and can always say why, or say "not ours."

A classifier that just saw "pothole" would have booked a Montréal work order for another city's street.
Resolver → redirect to Ville de Mont-RoyalIn the City's data, interior Mont-Royal requests are never assigned to borough road work ✓
Mont-Royal is an enclave — a separate municipality entirely surrounded by Montréal. The pin lands inside its boundary, so the answer is "not ours" no matter what the photo shows. Geometry alone gets this right only if it knows the enclave exists; a category classifier never does.
