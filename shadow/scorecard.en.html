The frozen policy

A lookup table fit on 961,599 requests created before 2025-01-01 (2024 & earlier) — 841 categories, 125 of them routed to a fixed shared-service unit, the rest by geography. Its sha256, minted at the freeze and unchanged since:

b1f8107035578539f3c3a9f81e2ec1d9365ec038ed409ab8fc1bf96ea915a1f1
Why you can trust this number without trusting us A prediction is a deterministic function of this frozen table and a request's category + geographic territory — nothing else. There are no model weights, no free parameters, nothing hidden. Recompute any prediction yourself from the same public inputs and you get the same answer. The table was hashed before the tested requests existed, so it cannot have been fit to them. Each daily run is committed to git; the commit history is the proof-of-time.
What this scorecard does not claim It does not claim to predict a routing before the City makes it — the 311 open data publishes each request already routed. The claim is narrower and checkable: a provably-not-retrofitted policy, independently recomputable, that keeps reproducing the City's own routing on requests it never saw. It scores the routing resolver only, not photo/voice perception.

Reproduce it

node shadow/shadow-run.mjs --as-of 2026-07-22 — pulls the live City API, re-derives every figure, and fails loudly if the committed policy hash no longer matches its own rules.
